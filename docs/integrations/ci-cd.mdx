---
title: "CI/CD integration (application-owned)"
description: "Use the LyraShield application boundary"
---

LyraShield Engine supports deterministic repository CI for its own source and a headless executable for the controlled worker. It is not distributed as a generic customer CI installer.

The application boundary provides queue admission, tenant authorization, target ownership, model routing, protected budgets, cancellation, evidence storage, deterministic scanners, result manifests, and reporting. Bypassing it would discard those controls.

For engine development, run:

```bash
bash scripts/verify-thin-fork.sh
git diff --check
```

For product CI/CD usage, follow the current LyraShield AI application documentation. Do not copy retained upstream Strix installer examples.
