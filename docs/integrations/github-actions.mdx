---
title: "GitHub Actions (application-owned)"
description: "LyraShield Engine is not installed directly in customer workflows"
---

<Warning>
The former upstream installer workflow is unsupported. Do not curl an upstream installer, invoke a nonexistent `strix` executable, or place model credentials in customer repositories.
</Warning>

The LyraShield application owns GitHub integration, diff scoping, approvals, queue admission, worker credentials, artifact parsing, evidence state, and SARIF/report behavior. Engine CI in this repository verifies the package and public worker contract; it does not provide a customer-facing scan action.

Any future direct engine action must be a separate reviewed product decision with pinned artifacts, least-privilege credentials, protected budgets, cancellation, output schema validation, and truthful evidence semantics.
