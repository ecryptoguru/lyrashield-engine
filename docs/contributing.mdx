The root [CONTRIBUTING.md](https://github.com/ecryptoguru/lyrashield-engine/blob/main/CONTRIBUTING.md) is authoritative.

## Required workflow

1. Branch from current `main`; never push directly to `main`.
2. Keep generic upstream sandbox/tool/SDK plumbing close to the pinned Strix release.
3. Put LyraShield model, budget, lifecycle, identity, evidence, and artifact behavior behind explicit reviewed boundaries.
4. Preserve the LyraShield modification banner on changed upstream Python files.
5. Add tests and update `NOTICE`, `UPGRADES.md`, and operator docs when the contract changes.
6. Run `bash scripts/verify-thin-fork.sh` and `git diff --check`.
7. Require human approval and green Engine CI.

<Warning>
Do not add providers or models outside GPT-5.6 Sol, Terra, and Luna; re-enable telemetry; weaken budget reservations; persist raw model output; or make confidence equivalent to verification.
</Warning>

Artifact schema changes require coordinated worker compatibility testing against `ecryptoguru/lyrashield-ai`.
