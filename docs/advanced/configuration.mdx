title: "Configuration"
description: "Supported LyraShield Engine environment variables"
---

Production configuration is owned by the LyraShield worker. The `lyrashield` adapter maps supported `LYRASHIELD_*` names to inherited internal settings and forces telemetry off.

## Model and endpoint

<ParamField path="LYRASHIELD_LLM" type="string" required>
  GPT-5.6 Sol, Terra, or Luna deployment name. Maps to internal `STRIX_LLM`. Other model families are rejected before sandbox setup.
</ParamField>

<ParamField path="LYRASHIELD_DELEGATE_LLM" type="string">
  Optional GPT-5.6 deployment for child specialists. The application supplies Luna for Deep/Custom scans and otherwise uses the primary deployment.
</ParamField>

<ParamField path="LLM_API_KEY" type="string">
  Credential for the approved OpenAI-compatible endpoint. Azure aliases are listed below.
</ParamField>

<ParamField path="LLM_API_BASE" type="string">
  Base URL for the approved endpoint.
</ParamField>

<ParamField path="LLM_API_VERSION" type="string">
  API version for endpoints that require it.
</ParamField>

<ParamField path="LYRASHIELD_REASONING_EFFORT" default="medium" type="string">
  Maps to internal reasoning effort. The application resolves medium for every scan mode.
</ParamField>

<ParamField path="LYRASHIELD_DELEGATE_REASONING_EFFORT" default="medium" type="string">
  Reasoning effort for child specialists. Production keeps this at medium.
</ParamField>

<ParamField path="LLM_TIMEOUT" default="300" type="integer">
  Model request timeout in seconds.
</ParamField>

Azure compatibility aliases:

- credentials: `AZURE_AI_API_KEY`, `AZURE_OPENAI_API_KEY`;
- endpoint: `AZURE_AI_API_BASE`, `AZURE_OPENAI_ENDPOINT`, `AZURE_OPENAI_API_BASE`;
- API version: `AZURE_API_VERSION`, `AZURE_OPENAI_API_VERSION`.

## Sandbox

<ParamField path="LYRASHIELD_IMAGE" type="string">
  Reviewed sandbox image reference. Production should use an inspected digest, not a floating tag.
</ParamField>

<ParamField path="LYRASHIELD_RUNTIME_BACKEND" default="docker" type="string">
  Sandbox runtime backend. The current worker contract uses Docker.
</ParamField>

<ParamField path="LYRASHIELD_MAX_LOCAL_COPY_MB" default="1024" type="integer">
  Maximum local directory size copied into the sandbox. Use a read-only `--mount` for larger operator-owned repositories.
</ParamField>

<ParamField path="STRIX_SANDBOX_ALLOW_HOST_GATEWAY" default="0" type="boolean">
  Opt-in host-gateway access. Keep disabled unless an explicitly authorized local workflow requires it. This is not a substitute for deployment egress policy.
</ParamField>

## Telemetry and artifacts

`LYRASHIELD_TELEMETRY` is accepted only as a compatibility alias; the adapter unconditionally sets inherited `STRIX_TELEMETRY=0`. Do not configure remote Traceloop/PostHog/Scarf export for production scans.

Bounded local artifacts are written to `strix_runs/<run-name>/`. The application worker consumes `run.json` and `vulnerabilities.json`, does not persist raw stdout/stderr, and owns evidence storage outside the engine work directory.

## Config file

The inherited CLI supports `--config <path>` for a JSON environment override. Production should prefer explicit worker-owned environment variables so model, credential, sandbox, and limit provenance remains auditable.

<Warning>
Do not place credentials, repository secrets, raw provider payloads, or user-controlled endpoints in committed config files. Configuration compatibility does not authorize a target or a paid scan.
</Warning>
