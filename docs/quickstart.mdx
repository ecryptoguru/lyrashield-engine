## Prerequisites

- Python 3.12+
- Docker running locally
- [uv](https://docs.astral.sh/uv/)
- a reviewed sandbox image
- for an authorized scan only, an approved GPT-5.6 Sol, Terra, or Luna endpoint

## Install from source

```bash
git clone https://github.com/ecryptoguru/lyrashield-engine.git
cd lyrashield-engine
uv sync --frozen
uv run lyrashield --version
uv run lyrashield --help
```

There is no supported `strix` executable or `strix.ai` installer in this distribution. Production builds the sibling engine source into the LyraShield worker image.

## Configure GPT-5.6

```bash
export LYRASHIELD_LLM="openai/gpt-5.6-luna"
export LYRASHIELD_REASONING_EFFORT="medium"
export LLM_API_KEY="<credential>"
export LLM_API_BASE="https://<approved-endpoint>"
```

Azure deployments can use the aliases documented in [Azure configuration](/llm-providers/azure). Provider credentials are optional only when the approved endpoint supplies authentication by another controlled mechanism.

## Run an authorized repository scan

```bash
uv run lyrashield \
  --target ./approved-repository \
  --scan-mode quick \
  --non-interactive \
  --max-budget-usd 1.20
```

<Warning>
The CLI accepts several upstream target forms, but the LyraShield production contract invokes it only for repository targets. Do not use direct CLI URL scanning as evidence of the application's pinned URL-scanner behavior.
</Warning>

Results are written under `strix_runs/<run-name>/`. Treat engine artifacts as untrusted input and consume them through the application worker contract.

## Verify without a paid scan

```bash
bash scripts/verify-thin-fork.sh
git diff --check
```

The legacy script name is retained for automation compatibility. The repository is maintained as a controlled derivative.
