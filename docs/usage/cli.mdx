## Basic usage

```bash
lyrashield (--target <target> | --target-list <path> | --mount <path>) [options]
```

Production invokes `lyrashield`, not `strix`, and supplies `--non-interactive`, a stable `--run-name`, and a positive `--max-budget-usd`.

## Target and scope

| Option | Meaning |
| --- | --- |
| `--target`, `-t` | Target value; repeatable. Production uses repository targets only. |
| `--target-list` | File with one non-empty, non-comment target per line. |
| `--mount` | Read-only local-directory bind mount; repeatable. It is convenience, not a hard isolation boundary. |
| `--instruction` | Bounded custom scan instruction. |
| `--instruction-file` | File containing custom instructions. |
| `--scope-mode` | `auto`, `diff`, or `full` for code scope. |
| `--diff-base` | Base branch or commit used for diff scope. |

Remote repositories are cloned at scan time. Local copy-size preflight does not bound remote repository size. The worker applies its own owned-path, timeout, and cleanup controls.

## Execution

| Option | Meaning |
| --- | --- |
| `--scan-mode`, `-m` | `quick`, `standard`, or `deep`; CLI default is `deep`. |
| `--non-interactive`, `-n` | Disable the TUI; required for worker/CI execution. |
| `--max-budget-usd` | Positive whole-run spend limit with concurrency-safe pre-request reservations. |
| `--run-name` | Stable orchestrator-owned run identifier. |
| `--resume` | Resume a prior run by run name. The application does not silently resume or replay orphaned scans. |
| `--config` | JSON config override path. Prefer worker-owned environment configuration in production. |
| `--version`, `-v` | Print LyraShield Engine package version. |

Pre-request reservations bound concurrent aggregate spend, but provider metering and in-flight behavior can still differ from local estimates. The application retains provider reconciliation separately and treats the approved cap as the billing-state ceiling.

## Examples

```bash
# Deterministic package/CLI checks; no model call
uv run lyrashield --version
uv run lyrashield --help

# Authorized local repository scan
uv run lyrashield --target ./repository --scan-mode quick --non-interactive --max-budget-usd 1.20

# Explicit diff scope
uv run lyrashield --target ./repository --scan-mode quick --scope-mode diff --diff-base origin/main --non-interactive --max-budget-usd 1.20
```

<Warning>
CLI support for a target form is not the LyraShield application contract. URL targets bypass the engine in production and use the pinned deterministic URL scanner.
</Warning>
