---
title: "Custom Instructions"
description: "Add bounded context to an authorized scan"
---

Use `--instruction` or `--instruction-file` to clarify an authorized repository's architecture, expected entry points, or approved focus areas.

```bash
uv run lyrashield \
  --target ./approved-repository \
  --instruction "Review authorization boundaries in the account API; do not access external services" \
  --scan-mode quick \
  --non-interactive \
  --max-budget-usd 1.20
```

Instructions are untrusted model-facing input. They remain subject to target validation, prompt-injection defenses, context/output limits, sandbox policy, and the worker's evidence rules. Instructions cannot:

- authorize a target or expand scope;
- enable unsupported providers or host-gateway access;
- override model, reasoning, budget, output, agent, cancellation, or cleanup controls;
- declare a finding verified or suppress immutable evidence-state rules.

Never embed production passwords, API keys, tokens, customer data, or secrets in instructions. Use explicitly provisioned, least-privilege test credentials through the controlled deployment mechanism when authenticated testing is approved.
