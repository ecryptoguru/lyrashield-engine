---
title: "GPT-5.6 Endpoints"
description: "Supported model and endpoint boundary"
---

LyraShield Engine accepts only deployment names recognized as GPT-5.6 Sol, Terra, or Luna. LiteLLM remains an internal model-client dependency; it does not make its broader provider catalog a supported LyraShield surface.

| Route | Supported use |
| --- | --- |
| OpenAI-compatible | Approved GPT-5.6 deployment with `LLM_API_KEY` and optional `LLM_API_BASE` |
| Azure AI / Azure OpenAI-compatible | Approved GPT-5.6 deployment with Azure endpoint, credential, and API-version aliases |

Anthropic, Bedrock, Vertex, OpenRouter, Novita, local models, Perplexity, and Parallel are unsupported and rejected or outside the production contract.

The application worker resolves the preset before spawning the engine: Safe/Quick/Standard use Luna with medium reasoning; Deep/Custom use Terra with high reasoning. `LYRASHIELD_LLM` remains the explicit fallback. This is mode routing, not an automatic Luna-to-Terra cascade within one scan.
