---
title: "OpenAI-compatible"
description: "Configure an approved GPT-5.6 endpoint"
---

```bash
export LYRASHIELD_LLM="openai/gpt-5.6-luna"
export LLM_API_KEY="<credential>"
export LLM_API_BASE="https://<approved-endpoint>"
```

`LLM_API_BASE` may be omitted when the approved client default is correct. A compatible API shape is not sufficient by itself: the model/deployment name must identify GPT-5.6 Sol, Terra, or Luna.

Use `LYRASHIELD_REASONING_EFFORT` only for operator-controlled local execution. The application worker owns reasoning selection in production.

Never commit credentials or place them in scan instructions. The worker passes only the bounded OpenAI/Azure credential surface to the engine subprocess.
