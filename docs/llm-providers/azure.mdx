---
title: "Azure-compatible"
description: "Configure GPT-5.6 through Azure AI or Azure OpenAI"
---

Use an Azure deployment whose configured name identifies GPT-5.6 Sol, Terra, or Luna.

```bash
export LYRASHIELD_LLM="azure/gpt-5.6-luna"
export AZURE_AI_API_KEY="<credential>"
export AZURE_AI_API_BASE="https://<approved-resource-endpoint>"
export LLM_API_VERSION="<approved-api-version>"
```

The settings layer also accepts these compatibility aliases:

| Purpose | Accepted variables |
| --- | --- |
| Credential | `AZURE_AI_API_KEY`, `AZURE_OPENAI_API_KEY`, `LLM_API_KEY` |
| Endpoint | `AZURE_AI_API_BASE`, `AZURE_OPENAI_ENDPOINT`, `AZURE_OPENAI_API_BASE`, `LLM_API_BASE` |
| API version | `LLM_API_VERSION`, `AZURE_API_VERSION`, `AZURE_OPENAI_API_VERSION` |

Use one coherent route per deployment. Do not mix endpoints and credentials from different Azure resources. The application worker, not end-user input, selects Luna or Terra and the reasoning effort in production.
